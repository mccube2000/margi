# src 模块注意事项

## 使用约束

- 不要直接修改 `Module` 结构体的字段，必须通过 `ModuleManager` 提供的接口进行操作
- 不要绕过状态机直接设置模块状态，必须调用相应的状态更新函数
- 不要手动修改 `.margi/config.json`，使用命令行工具进行管理

## 已知的坑

- 修改 `Command` 枚举时必须同步更新所有相关的命令处理函数
- 添加新命令时需要在 `run()` 函数中添加对应的处理分支
- 文档路径变更时必须更新配置文件，否则会导致文档丢失

## 耦合关系

- 修改 `ModuleStatus` 枚举会影响所有状态相关的逻辑
- 更改 `MargiError` 类型需要更新所有错误处理代码
- 修改模块结构会影响所有子模块的文档生成逻辑

## 性能注意事项

- 大目录扫描时注意内存使用，避免一次性加载所有文件
- 文档生成是 I/O 密集型操作，注意并发控制
- 搜索功能依赖索引构建，首次使用时会有延迟

## 安全注意事项

- 所有文件路径都必须经过验证，防止路径遍历攻击
- 用户输入的命令参数需要严格校验
- 配置文件读写需要处理权限问题
