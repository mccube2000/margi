# src/module 模块注意事项

## 使用约束

- 不要直接修改配置文件中的模块信息，必须通过 `ModuleManager` 接口
- 不要手动改变模块状态，让系统自动管理状态转换
- 注册模块时必须使用绝对路径，避免相对路径问题

## 已知的坑

- 修改 `ModuleStatus` 枚举时必须更新所有状态转换逻辑
- 模块路径变更时必须更新配置文件，否则模块会丢失
- 文件系统权限不足时会导致扫描失败，需要检查访问权限

## 耦合关系

- 修改 `Module` 结构体会影响所有相关的存储和加载逻辑
- 更改配置文件格式需要同时更新序列化和反序列化代码
- 状态转换逻辑变更会影响所有依赖状态的模块操作

## 性能注意事项

- 大项目扫描时注意内存使用，避免一次性加载所有文件路径
- 频繁的状态检查会增加 I/O 开销，建议使用缓存
- 模块数量多时，列表操作的性能需要注意

## 边界情况

- 跨平台路径分隔符需要统一处理（Windows 使用 \，Unix 使用 /）
- 符号链接可能导致无限循环，需要特殊处理
- 模块名称中包含特殊字符可能影响文件系统操作

## 最佳实践

- 模块名称应该简洁且有意义，避免使用空格和特殊字符
- 定期清理不再使用的模块，避免配置文件膨胀
- 使用版本控制管理模块配置的变更

## 设计约束

- 模块名称必须是唯一的，不能重复注册
- 模块路径必须是有效的目录，不能指向文件
- 文档目录必须与模块路径分离，避免污染源码
