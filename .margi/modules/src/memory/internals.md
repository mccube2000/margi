# src/memory 模块内部实现

## 核心数据流

### 笔记处理流程
```
添加笔记
  ↓
验证内容
  ↓
生成 UUID
  ↓
存储到文件系统
  ↓
更新索引
```

### 纠正记录流程
```
添加纠正
  ↓
验证格式
  ↓
存储到纠正目录
  ↓
更新全局纠正索引
```

## 关键设计决策

### 1. 文件存储架构
- 笔记存储在 `.margi/memory/notes/` 目录下
- 纠正记录存储在 `.margi/memory/corrections/` 目录下
- 每个模块有独立的子目录，避免冲突

### 2. UUID 唯一标识
- 使用 UUID v4 作为记录 ID，确保全局唯一性
- 避免使用自增 ID，便于分布式使用
- ID 包含在文件名中，便于检索

### 3. 时间戳管理
- 使用 UTC 时间戳，避免时区问题
- 所有时间相关操作都使用 `DateTime<Utc>`
- 文件名中包含时间戳，便于按时间排序

## 性能优化

### 文件操作
- 使用异步文件 I/O 提高并发性能
- 批量写入减少磁盘 I/O 次数
- 内存缓存频繁访问的记录

### 索引管理
- 维护内存中的索引结构
- 定期持久化索引到磁盘
- 使用高效的查找算法
