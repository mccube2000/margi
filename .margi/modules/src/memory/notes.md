# src/memory 模块注意事项

## 使用约束

- 不要直接创建文件来存储笔记，必须通过 `MemoryManager` 提供的接口
- 不要手动修改笔记文件的内容，会导致数据不一致
- 添加纠正记录时必须提供明确的错误做法和正确做法

## 已知的坑

- 修改笔记内容后必须重新生成索引，否则搜索不到最新内容
- 删除笔记文件时必须同步更新内存索引
- 文件名中的时间戳必须使用 UTC 格式，否则排序会出错

## 耦合关系

- 修改 `Note` 结构体会影响所有相关的存储和检索逻辑
- 更改存储目录结构需要同时更新内存索引构建逻辑
- 纠正记录的格式变更会影响所有读取纠正记录的代码

## 性能注意事项

- 大量笔记时注意内存使用，考虑使用分页加载
- 文件系统操作是同步的，避免在热路径中进行频繁的文件 I/O
- 搜索功能会遍历所有文件，对于大量数据建议建立索引

## 边界情况

- 磁盘写满时写入操作会失败，需要处理 `std::io::Error`
- 并发写入同一模块的笔记可能导致文件名冲突
- 跨平台文件路径分隔符需要统一处理

## 最佳实践

- 笔记内容不要太长，避免单文件过大
- 及时清理过期的纠正记录，避免存储空间浪费
- 定期备份数据目录，防止数据丢失
